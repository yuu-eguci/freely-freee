### holiday_type:full について有給の申請がある日については、勤怠の登録方法をこうする

```bash
curl --location --request PUT 'https://api.freee.co.jp/hr/api/v1/employees/<employee_id>/work_records/<date>-12' \
--header 'Accept: application/json' \
--header 'Authorization: Bearer <token>' \
--data '{
  "company_id": <company_id>,
  "paid_holidays": [
    {
      "type": "full"
    }
  ]
}'
```

### holiday_type:half について有給の申請がある日については、勤怠の登録方法をこうする

```bash
{
  "company_id": {{FREEE_COMPANY_ID}},
  "work_record_segments": [
    {
      // デフォルトである 09:00 - 19:00 から、 half で申請されている時間を引いたものを登録する。もし 09:00 - 19:00 をふたつに分けるような変な申請だった場合は、デフォルトで登録して、ログに警告コメントを出す。
      "clock_in_at": "2026-03-31 14:00:00",
      "clock_out_at": "2026-03-31 19:00:00"
    }
  ],
  "paid_holidays": [
    {
      "type": "half",
      "mins": 240
    }
  ]
}
```

### 所感

- というわけで、これまではこう↓だったが……
    - 平日 -> 09:00 - 19:00 で勤怠登録
    - 休日 -> 勤怠登録なし
- これからはこう↓になる。
    - 平日で有給申請なし -> 09:00 - 19:00 で勤怠登録
    - 平日で有給申請あり(holiday_type:full) -> 有給として勤怠登録
    - 平日で有給申請あり(holiday_type:half) -> 半休として勤怠登録
    - 休日 -> 勤怠登録なし

## 詳細設計

### 1. この設計で追加すること

既存の「指定月の平日一括勤怠登録」に対して、対象月の有給申請 (`paid_holidays`) を先に取り込み、日付ごとに以下の分岐で `work_records` の PUT payload を切り替える。

- 平日かつ有給なし: 既存どおり `09:00-19:00` 登録
- 平日かつ `holiday_type=full`: `paid_holidays=[{"type":"full"}]` で登録
- 平日かつ `holiday_type=half`: `09:00-19:00` から申請時間を差し引いた勤務帯 + `paid_holidays=[{"type":"half","mins":...}]` で登録
- 休日: 既存どおりスキップ

### 2. API 呼び出し順

1. 対象月 (`yyyy-mm`) を入力
2. `GET /hr/api/v1/users/me` で `company_id` と `employee_id` を確定
3. `GET /hr/api/v1/approval_requests/paid_holidays` で対象月の有給申請を取得
4. 対象月の日付を 1 日ずつ処理
5. 各日で `GET /hr/api/v1/employees/{employee_id}/work_records/{date}` を取得して対象日判定
6. 対象日のみ `PUT /hr/api/v1/employees/{employee_id}/work_records/{date}` を実行

補足:

- `paid_holidays` は月初に 1 回取得し、`target_date` をキーに辞書化して日次処理で再利用する
- `paid_holidays` の API は `total_count` を見て取り切る。`limit` を固定し、未取得があれば次ページを取得する

### 3. 対象日の判定ルール (平日/休日)

対象日の最終判定はカレンダー曜日ではなく、`work_records/{date}` のレスポンス値を正とする。

- 実行対象条件:
  - `day_pattern == "normal_day"`
  - `use_default_work_pattern == true`
- 上記どちらかを満たさない日はスキップ

スキップログ例:

- `[SKIP] 2026-03-01 reason=day_pattern_not_normal_day detail=day_pattern=prescribed_holiday`
- `[SKIP] 2026-03-10 reason=use_default_work_pattern_false detail=use_default_work_pattern=false`

### 4. 有給申請データの採用ルール

1 日につき利用する有給申請は 0 件または 1 件に正規化する。

- 採用対象:
  - `target_date` が当日一致
  - `revoke_status == null`
  - `holiday_type` が `full` または `half`
- 優先順位:
  - `full` が 1 件でもあれば `full` 優先
  - `full` がなく `half` が 1 件なら `half` 採用
  - 上記以外 (同日複数 `half`、不正データ混在など) はフォールバック扱い

### 5. half の時間差分ロジック

基準勤務帯は固定で `09:00-19:00`。
`half` の `start_at` / `end_at` を当日日時へ展開し、差分を 1 本の `work_record_segments` に落とす。

- 午前半休パターン:
  - 条件: `start_at == 09:00` かつ `09:00 < end_at <= 19:00`
  - 勤務帯: `end_at - 19:00`
  - `mins`: `end_at - 09:00` (分)
- 午後半休パターン:
  - 条件: `end_at == 19:00` かつ `09:00 <= start_at < 19:00`
  - 勤務帯: `09:00 - start_at`
  - `mins`: `19:00 - start_at` (分)

`half` 採用時の payload 例:

```json
{
  "company_id": 123,
  "work_record_segments": [
    {
      "clock_in_at": "2026-03-31 14:00:00",
      "clock_out_at": "2026-03-31 19:00:00"
    }
  ],
  "paid_holidays": [
    {
      "type": "half",
      "mins": 300
    }
  ]
}
```

### 6. フォールバック条件

次のいずれかに該当した `half` は「変則 half」とみなし、有給連携を使わず通常勤務登録へフォールバックする。

- `start_at` / `end_at` が null
- `start_at >= end_at`
- 申請時間が `09:00-19:00` の外に出る
- 申請時間が勤務帯の中央にあり、差分が 2 セグメント必要になる
- 同日に `half` が複数あり一意に決められない

フォールバック時の動き:

- payload は「有給なし」の既定 (`09:00-19:00`) を使用
- `paid_holidays` は送らない
- 警告ログを出す

警告ログ例:

- `[WARN] 2026-03-31 reason=half_fallback detail=request_id=16080378,start_at=10:00,end_at=15:00`

### 7. エラー時方針

- 月初の前処理 (`users/me`, `paid_holidays` 取得) が失敗したら、その時点で処理中断して終了コード `1`
- 日次処理中に `work_records` の GET/PUT が失敗したら、当日エラーを出して即中断 (既存方針を維持)
- 異常系ログは必ず API ステータスコードとレスポンスの `errors[].messages[]` を出力

エラーログ例:

- `[ERR] 2026-03-12 step=put_work_record status=400 message=給与明細が確定されているので、更新できません。`

### 8. 将来拡張ポイント

- 変則 `half` を 2 セグメント勤務として正しく登録する拡張
- `status` (approved / in_progress など) の採用条件を設定値化
- 半休時の休憩 (`break_records`) 自動計算ルールを追加
- 有給日 (`full`/`half`) のときの勤怠タグ付与ルールを明示的に分岐
