# 013 機能名変更設計

## 要望

メニューおよびコード内の機能名を以下のとおり変更する。

- "指定の月の平日に勤怠を詰め込む" -> "指定の月に勤怠を詰め込む"
- "指定月の勤怠をリセットする" -> "指定の月の勤怠をリセットする"

## 詳細設計

### 変更が必要なファイル一覧

ソースコード:

- `app/actions/registry.py`

設計ノート (記録としての文言が残っているが、過去ノートなので変更しない):

- `docs/agent/006-指定月の平日一括勤怠登録設計.md` — 過去設計の記録なので変更しない
- `docs/agent/007-指定月の平日一括勤怠登録の除外条件追加設計.md` — 同上
- `docs/agent/009-メニュー選択UIの左揃え修正設計.md` — 同上
- `docs/agent/010-勤怠リセットを全日対象へ修正設計.md` — 同上
- `docs/agent/011-CLI出力インデント崩れ修正設計.md` — 同上
- `docs/agent/012-指定月平日一括勤怠登録に有給申請対応を追加設計.md` — 同上

### 変更箇所の具体的な文字列

`app/actions/registry.py`:

```
# before
menu_label="指定の月の平日に勤怠を詰め込む",

# after
menu_label="指定の月に勤怠を詰め込む",
```

```
# before
menu_label="指定月の勤怠をリセットする",

```

### 影響範囲の確認

- メニュー表示: `registry.py` の `menu_label` がそのままメニューに出るので、ここだけ変えれば OK
- `action_id` (`bulk_attendance`, `bulk_attendance_reset`) は変更なし — ルーティングには影響なし
- `app/actions/bulk_attendance.py` 内のコメントや変数名に旧文言があるか確認が必要だが、今回の grep 結果では該当なし
- 過去設計ノートに旧文言が多数残っているが、それらは当時の記録なので変更不要

## レビュー

### 指摘1: `bulk_attendance_reset.py` の docstring が変更対象から漏れている

「影響範囲の確認」では `bulk_attendance.py` のみに言及しているが、 `bulk_attendance_reset.py` にも旧文言を含む docstring が 2 箇所ある。変更が必要なファイル一覧に `bulk_attendance_reset.py` が抜けてる。

実際のソース:

```python
# app/actions/bulk_attendance_reset.py:1
"""指定月の勤怠を一括でリセットするアクションです。"""

# app/actions/bulk_attendance_reset.py:19
    """指定月の勤怠を一括でリセットします。"""
```

要望が「コード内の機能名を変更する」とある以上、コード内コメントも変更対象に含まれるはず。これらも "指定の月の勤怠" に揃えるべき。

### 指摘2: `bulk_attendance.py` の docstring も要確認 (記述が不正確)

設計ノートに「今回の grep 結果では該当なし」と書いてあるが、実際に grep すると `bulk_attendance.py` にも旧文言がある。

```python
# app/actions/bulk_attendance.py:1
"""指定月の平日に一括で勤怠を登録するアクションです。"""

# app/actions/bulk_attendance.py:42
    """指定月の平日に一括で勤怠を登録します。"""
```

"指定月の平日" という表記なので、新しい機能名 "指定の月に勤怠を詰め込む" に合わせるなら "指定の月の平日に" か "指定の月に" へ変えることを検討すべき。

少なくとも「grep 結果では該当なし」という記述は事実と異なるので修正が必要。

### 指摘3: 変更が必要なファイル一覧の見直しを

上記を踏まえて、以下が必要:

- `app/actions/bulk_attendance_reset.py` を変更対象に追加する
- `app/actions/bulk_attendance.py` の方針 (変更する/しない) を明示する
- 「今回の grep 結果では該当なし」という誤った記述を削除または訂正する
