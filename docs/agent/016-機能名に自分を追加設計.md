# 016 機能名に「自分」を追加する変更設計

## 要望

- "指定の月に勤怠を詰め込む" -> "指定の月に自分の勤怠を詰め込む" に機能名変更
- "指定の月の勤怠をリセットする" -> "指定の月の自分の勤怠をリセットする" に機能名変更

## 詳細設計

- あたしの今回のゴールは、CLI メニューに出す機能名だけを「自分の勤怠」表記にそろえること。
- 相棒向けに先に結論を書くと、`app/` 配下で変更必須なのは `app/actions/registry.py` の 2 箇所。

### 1. 変更対象の洗い出し ( ソースコード )

- `app/actions/registry.py`
    - `action_id="bulk_attendance"` の `menu_label` を `指定の月に自分の勤怠を詰め込む` へ変更。
    - `action_id="bulk_attendance_reset"` の `menu_label` を `指定の月の自分の勤怠をリセットする` へ変更。
- それ以外の `app/` 配下ファイルは、今回の機能名文字列を直接保持していないため変更対象外。
- `app/actions/bulk_attendance.py` と `app/actions/bulk_attendance_reset.py` は処理説明の docstring のみを持っていて、今回要望の「機能名変更」とは対象を分ける。

### 2. 変更対象の洗い出し ( 設計ノート )

- `docs/agent/016-機能名に自分を追加設計.md`
    - このファイルの `詳細設計` セクションに、変更箇所とテスト方針を追記。
- 過去設計ノート ( `013` `014` `015` など ) は履歴として固定し、今回タスクの更新対象にしない。

### 3. テスト変更箇所の設計

- 現状の `tests/` 配下は `test_bulk_attendance_paid_holidays.py` のみで、メニュー表示名を検証するテストがない。
- 機能名変更の回帰防止として、`app.actions.registry.to_menu_items( debug_mode=False )` の戻り値ラベルを検証するテストを追加対象にする。
- 追加候補ファイルは `tests/test_action_registry_menu_labels.py`。
- 想定テストケースは次の 3 つ。
    - ラベル一覧に `指定の月に自分の勤怠を詰め込む` が含まれること。
    - ラベル一覧に `指定の月の自分の勤怠をリセットする` が含まれること。
    - ラベル一覧に旧名称 2 つ ( `指定の月に勤怠を詰め込む` と `指定の月の勤怠をリセットする` ) が含まれないこと。

### 4. 影響範囲

- 影響範囲はメニュー表示テキストのみ。
- `action_id` とハンドラ実装は不変なので、実行フローと freee API 呼び出しロジックへの影響はない。
