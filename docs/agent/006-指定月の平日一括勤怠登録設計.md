## 要望

### 新しい選択肢追加 "指定の月の平日に勤怠を詰め込む for yyyy-mm"

以下のような順番で処理

### コマンドライン引数で月を取得

`yyyy-mm`

### アクセストークン取得

これは実装済み。

### employee id, company id 取得

curl で動作確認済み。

```bash
curl --location 'https://api.freee.co.jp/hr/api/v1/users/me' \
--header 'Accept: application/json' \
--header 'Authorization: Bearer <TOKEN>'
```

```json
{
  "id": 111,
  "companies": [
    {
      "id": 222, // ここが company_id
      "name": "...",
      "role": "self_only",
      "external_cid": "333",
      "employee_id": 444, // ここが employee_id
      "display_name": "..."
    }
  ]
}
```

### 勤怠タグ id 取得

```bash
curl --location 'https://api.freee.co.jp/hr/api/v1/employees/<employee_id>/attendance_tags?company_id=<company_id>' \
--header 'Accept: application/json' \
--header 'Authorization: Bearer <TOKEN>'
```

```json
{
  "employee_attendance_tags": [
    {
      "id": 555, // ここが勤怠タグ id
      "company_id": 222,
      "name": "出社（事務所／常駐先）", // "出社" と入っているものを取得する。
      "description": "テレワークと出社と混在する場合、出社日にタグを付与してください。",
      "max_amount": 1,
      "published": true,
      "is_employee_usable": true
    }
  ]
}
```

### この月のすべての日付に対して以下を行う

curl で動作確認済み。

```bash
curl --location 'https://api.freee.co.jp/hr/api/v1/employees/<employee_id>/work_records/<date>?company_id=<company_id>' \
--header 'Accept: application/json' \
--header 'Authorization: Bearer <TOKEN>'
```

```json
{
  "date": "2026-02-21",
  "day_pattern": "normal_day", // ほか、 prescribed_holiday とかがある
  // ...
}
```
