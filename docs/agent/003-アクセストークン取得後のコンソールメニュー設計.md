# 003 アクセストークン取得後のコンソールメニュー設計

## 要望

- 002 ノートで設計した OAuth フローでアクセストークン取得が成功した直後に、次の行動を選べるようにする
- コンソールに「今回は何をしたい？」を表示し、選択肢を番号で入力できるようにする
- 最初に用意する選択肢は 1 つだけにする
  - `あ、いや、アクセストークン取得までいけるか見たかっただけ`
- 上記を選択したらアクセストークンを `print` して終了する

## 目的

- 「トークン取得まで動くことの確認」を、実行時に明示的に完了できるようにする
- 次の機能追加 (勤怠操作など) の前に、アプリの導線をメニュー化して拡張しやすくする

## 対象範囲

- 対象: `main.py` の実行フロー (認可コード交換成功時 / リフレッシュ成功時の直後)
- 非対象: 勤怠 API の呼び出し実装、メニューの複数機能実装

## 仕様

### メニュー表示条件

- アクセストークン取得が成功したら必ずメニューを表示する
  - 認可コードフロー成功時
  - リフレッシュフロー成功時

### メニュー表示内容

```text
今回は何をしたい？
1. あ、いや、アクセストークン取得までいけるか見たかっただけ
番号を入力してください: 
```

### 入力仕様

- `1` が入力された場合:
  - アクセストークンを標準出力へ表示
  - 終了コード `0` で終了
- それ以外が入力された場合:
  - 不正な入力としてエラーメッセージを表示
  - 終了コード `1` で終了
- 入力が空の場合も不正入力として扱う

### 出力仕様 (選択肢 1 の場合)

```text
access_token: {取得したアクセストークン}
```

## エラーハンドリング方針

- 002 ノートで定義した例外処理方針 (`ConfigError`, `TokenStoreError`, `OAuthTokenError`) は維持する
- 追加する入力エラーは既存の OAuth 例外と分離して扱う
  - 例: `MenuInputError` などの新しい例外を設ける、または `main()` で入力チェックして即エラー終了
- メニュー表示に到達する前の失敗時は、従来どおり認可 URL を表示して終了する

## 実装方針

1. アクセストークン取得処理 (`exchange_auth_code` / `refresh_access_token`) の返却値から `access_token` を取り出す
2. `select_action(access_token: str) -> int` のような関数を新設する
3. 関数内でメニュー表示・入力受付・分岐を実装する
4. `main()` の成功パスは直接終了せず、必ず `select_action` の戻り値を返す

## 想定実行例

```bash
$ python main.py --auth-code xxxxx
認可コードでトークンを取得し、token.json を更新しました。
今回は何をしたい？
1. あ、いや、アクセストークン取得までいけるか見たかっただけ
番号を入力してください: 1
access_token: eyJhbGciOi...
```
