# 007

## 要望

006 で設計した「指定の月の平日に勤怠を詰め込む」へ、以下の除外条件を追加する。

- 現在の除外条件:
  - `day_pattern != "normal_day"` は処理対象から外す
- 追加したい除外条件:
  - `use_default_work_pattern == false` も処理対象から外す

## 目的

- 会社や本人の個別勤務パターンが適用されている日 (`use_default_work_pattern == false`) には、自動投入を行わない
- 既存の normal_day 判定と合わせて、意図しない上書きを防ぐ

## 対象範囲

- `app/actions/bulk_attendance.py` の日次処理判定
- 設計ノート 006 の判定条件説明・ログ仕様の更新
- 必要ならスキップ件数サマリーの定義更新

## 非対象

- 認証フロー
- タグ付与ロジックの仕様変更
- 勤務時刻 (`09:00-19:00`, `11:30-12:30`) の変更

## 詳細設計

### 1. 追加仕様の要点

006 の日次判定を以下に拡張する。

- 既存: `day_pattern != "normal_day"` の日はスキップ
- 追加: `use_default_work_pattern == false` の日もスキップ

ねらいは「`normal_day` でも個別勤務パターン適用日には自動投入しない」こと。

### 2. 判定順序と判定条件

`GET /work_records/{date}` のレスポンスを使って、1 日ごとに次の順で判定する。

1. `day_pattern` 判定
   - 条件: `day_pattern != "normal_day"`
   - 結果: スキップ (`reason_code=day_pattern_not_normal_day`)
2. `use_default_work_pattern` 判定
   - 条件: `use_default_work_pattern == false`
   - 結果: スキップ (`reason_code=use_default_work_pattern_false`)
3. 上記 1, 2 を両方通過した日のみ、勤怠 PUT とタグ PUT を実行

判定順は 006 との整合を優先して `day_pattern` を先に評価する。  
この順にすることで、休日・所定休日などは従来どおり最初の条件で早期スキップできる。

### 3. スキップ理由のログ仕様

既存の `[SKIP]` 形式を維持しつつ、理由コードを明示する。

- 出力形式:
  - `[SKIP] <date> reason=<reason_code> detail=<key>=<value>`
- 例:
  - `[SKIP] 2026-03-01 reason=day_pattern_not_normal_day detail=day_pattern=prescribed_holiday`
  - `[SKIP] 2026-03-10 reason=use_default_work_pattern_false detail=use_default_work_pattern=false`

サマリーは既存 (`登録日数 / スキップ日数 / エラー日数`) を維持し、必要ならスキップ内訳を追加できる形にする。

- 追加候補:
  - `skip_day_pattern_count`
  - `skip_default_pattern_false_count`

### 4. 既存 006 との整合性

今回の追加は「スキップ条件の拡張」のみで、006 の主フローは維持する。

- 維持する項目
  - 実行順序 (`work_record` 取得 -> 判定 -> 勤怠 PUT -> タグ PUT)
  - エラー時即中断
  - 終了コードの扱い (`0 / 1 / 2 / 130`)
  - ログプレフィックス (`[OK] / [SKIP] / [ERR]`)
  - 0.5 秒待機の方針
- 更新する項目
  - 006 の「normal_day なら続行」の記述を「normal_day かつ `use_default_work_pattern != false` の場合に続行」へ更新
  - 006 のスキップログ例に `reason_code` を追加

### 5. 実装時の変更対象ファイル案

今回は設計追記のみ。実装フェーズでの変更対象は以下を想定する。

- `app/actions/bulk_attendance.py`
  - `_process_date()` のスキップ判定に `use_default_work_pattern == false` 条件を追加
  - `[SKIP]` ログの理由コード出力を追加
  - 必要に応じてサマリーのスキップ内訳カウントを追加
- `docs/agent/006-指定月の平日一括勤怠登録設計.md`
  - 判定条件・ログ仕様の記述を今回仕様に合わせて更新
- `docs/agent/007-指定月の平日一括勤怠登録の除外条件追加設計.md`
  - 本ノートを実装確定内容で追記・確定化
