# 014 有給登録時の出社タグ除外設計

## 要望

"指定の月に勤怠を詰め込む" 機能の修正。

    - 有給として勤怠登録するときは、出社タグを登録しない
    - 半休のときは現状のまま ( 出社タグを登録する ) で OK

## 詳細設計

相棒、あたしが今回の変更ポイントを実装フローに沿って固めるねｗ

### 1. 現在の実装フローの確認 ( `app/actions/bulk_attendance.py` )

`handler()` は月入力後に `users/me` と `attendance_tags` を取得し、日付ループで `_process_date()` を呼ぶ。

`_process_date()` の主要フローはこう。

    - `get_work_record()` で対象日を取得し、`day_pattern == "normal_day"` と `use_default_work_pattern == True` の日だけ処理する。
    - `_decide_paid_holiday()` で `decision.kind` を決める ( `none` / `full` / `half` / `half_fallback` ) 。
    - `decision` に応じた `work_record` payload を `_build_work_record_payload()` で作り、`put_work_record()` を実行する。
    - その後に常に `put_attendance_tags()` を実行して「出社」タグを付与している。

今は `decision.kind` が `full` でも `put_attendance_tags()` が呼ばれる実装になってる。

### 2. 有給登録と判定する条件 ( どの変数 / フラグを見るか )

有給登録かどうかの判定には `_process_date()` 内の `decision.kind` を使う。

    - `decision.kind == "full"` を「有給として勤怠登録する日」と定義する。
    - `decision.kind == "half"` は半休なので今回の除外対象にしない。
    - `decision.kind == "half_fallback"` は通常勤務へのフォールバックなので除外対象にしない。

`decision.kind` 自体は `_decide_paid_holiday()` が、`holiday_type` と `status` と `revoke_status` を見て決定しているため、今回のタグ除外条件は `decision.kind` だけで十分に判定できる。

### 3. 出社タグをスキップするための変更箇所

変更箇所は `_process_date()` の `put_work_record()` 成功後から `put_attendance_tags()` 呼び出し前。

    - 追加条件: `if decision.kind == "full":`
    - この条件のときは `put_attendance_tags()` を呼ばず、`[OK]` ログを出して `success` で return する。
    - 条件に入らないときだけ、既存どおり `tag_payload` を作って `put_attendance_tags()` を実行する。

この構成なら、勤怠登録 API ( `put_work_record` ) の成功を前提に、全休有給日だけタグ付与を確実にスキップできる。

### 4. 半休時の挙動 ( 変更なし ) の明記

半休の挙動は現状維持。

    - `decision.kind == "half"` の日は、これまでどおり `put_work_record()` 実行後に `put_attendance_tags()` を実行する。
    - `decision.kind == "half_fallback"` も現状どおりタグを付与する。
    - つまり「出社タグを外す」のは `full` の日だけ、という境界を固定する。

この境界でいけば、相棒の要望どおり「有給はタグなし、半休はタグあり」が素直に成立する♡
