# 009 メニュー選択 UI の左揃え修正設計

## このノートの位置づけ

- `001-このプロジェクトの概要書` の責務分離方針に従い、`menu` 層の描画責務だけを調整する
- 今回は設計のみを更新し、実装は行わない
- 操作仕様 (`↑` / `↓` / `Enter` / `Ctrl+C`) は既存どおり維持する

## 要望

選択肢 UI の表示が右側へずれて見えるため、選択中行・非選択行ともに左端起点で揃える。

現状イメージ:

```text
今回は何をしたい? (↑↓で選択 / Enterで決定 / Ctrl+Cで中断)
                                                         > 指定の月の平日に勤怠を詰め込む
                                                                                           指定月の平日勤怠をリセットする
```

期待する表示:

```text
今回は何をしたい? (↑↓で選択 / Enterで決定 / Ctrl+Cで中断)
> 指定の月の平日に勤怠を詰め込む
  指定月の平日勤怠をリセットする
```

## 対象範囲

- `app/menu/renderer.py` のメニュー再描画方式
- 左端基準の行描画ルール定義
- 既存 UI 操作仕様を維持したままの描画調整

## 非対象

- メニュー項目の増減・文言変更
- `app/menu/input_reader.py` のキー解釈仕様
- `app/menu/controller.py` の選択インデックス遷移仕様
- 認証・API・終了コード仕様

## 原因仮説

1. 再描画時のカーソル復帰が端末依存で不安定
2. `render_menu(initial=False)` が相対移動 (`\x1b[{len(items)}F`) 前提で、描画開始列が明示されていない
3. メニュー行の途中改行や端末幅制約が発生すると、次回再描画の基準行と基準列がずれやすい

補足:

- 現実装は概ね ANSI 対応端末で動作するが、`docker compose run --rm app` のように PTY 経路が変わると、左端基準が崩れて見えるケースがある
- 本件では「右にずれて見える」事象の再発防止を優先し、描画開始列を毎回明示する方針を取る

## 修正方針（レンダリング座標・カーソル制御）

### 1. 行の描画開始位置を毎回 `column 1` に固定する

- 各行描画前に `\r` または `\x1b[1G` を出力して、必ず左端から書き始める
- 行末は `\x1b[2K` または既存 `\x1b[K` でクリアし、前フレームの残骸を残さない

### 2. 再描画開始時に「メニュー先頭行」へ戻る手順を明確化する

- `initial=False` 時は、まず現在行先頭へ戻し、必要行数だけ上へ移動してメニューブロック先頭へ復帰する
- 「上移動だけ」「左移動だけ」に依存しないよう、縦移動と横位置リセットをセットで扱う

### 3. 選択行・非選択行の左端基準を統一する

- 選択行: `"> {label}"`
- 非選択行: `"  {label}"`
- どちらも 1 文字目から描画し、テキスト開始位置を揃える

### 4. 入力仕様は一切変更しない

- `↑` / `↓` / `Enter` / `Ctrl+C` の意味、選択遷移、終了コードマッピングは変更しない
- 変更は描画関数内の座標制御に限定する

## 変更対象ファイル案

1. `app/menu/renderer.py`（主変更）
   - カーソル復帰処理を左端固定前提へ更新
   - 行描画の共通ヘルパー（例: `_write_menu_line`）追加を検討
2. `app/menu/controller.py`（必要時のみ）
   - `render_menu(..., initial=...)` 呼び出し契約に変更が必要な場合のみ最小修正
3. `docs/agent/009-メニュー選択UIの左揃え修正設計.md`
   - 本設計の更新（今回実施）

## 既存挙動への影響範囲

- 影響あり:
  - メニューの見た目（左揃え、再描画の安定性）
- 影響なし（維持）:
  - キー操作仕様 (`↑` / `↓` / `Enter` / `Ctrl+C`)
  - 選択インデックス遷移ロジック
  - アクション実行分岐 (`registry` / `execute`)
  - 終了コード (`0 / 1 / 2 / 130`)
  - OAuth / token 保存 / API 呼び出し

## 確認観点（実装後チェックリスト）

1. ローカル実行 (`python main.py --auth-code ...`) で、初回表示から左端揃えになっている
2. `↑` / `↓` 押下ごとの再描画で、選択行・非選択行がともに左端から崩れない
3. `Enter` で従来どおり選択確定できる
4. `Ctrl+C` で従来どおり中断できる（終了コード `130`）
5. `docker compose run --rm app ...` 実行時でも表示ズレが再現しない
6. 複数回キー入力後も、文字の残り・右寄り表示・行ずれが発生しない

## レビュー観点

1. 変更が `menu` 層（主に `renderer`）へ閉じているか
2. 操作仕様・終了コード仕様に影響を与えていないか
3. Docker 経路を含めて左揃えが再現性高く維持されるか
