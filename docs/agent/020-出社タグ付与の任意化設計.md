# 020 出社タグ付与の任意化設計

## 要望

- 対象機能:
  - 「指定の月に自分の勤怠を詰め込む」
  - 「指定の月に従業員ID指定で勤怠を詰め込む」
- これら 2 機能は、現在は出社タグを PUT する。
- ただし不要な人もいるため、
  - 対象月入力
  - 出勤何時入力
  - 退勤何時入力
  の次に「出社タグいる？」を聞くようにする。
- デフォルトは「つけない」（Enter でつけない）。

## 詳細設計

- `app/actions/bulk_attendance.py` の入力フローに、タグ付与可否入力を追加する。
- 追加順序は既存要望どおり、`対象月 -> 出勤時刻 -> 退勤時刻 -> 出社タグ要否` とする。
- 入力仕様:
  - プロンプト: `出社タグいる？ (y/N。Enter でつけない): `
  - Enter: `False`（タグをつけない）
  - `y` / `yes`: `True`
  - `n` / `no`: `False`
  - それ以外: 入力エラー (`EXIT_CODE_MENU_ERROR=2`)
- 既存処理への反映:
  - `include_attendance_tag` フラグを実行関数へ渡す。
  - `include_attendance_tag=False` のときは、出社タグ API (`put_attendance_tags`) を呼ばない。
  - `include_attendance_tag=True` のときのみ、従来どおり出社タグ API を呼ぶ。
- 既存ルールとの整合:
  - `decision.kind == "full"`（有給 full）の日は、従来どおりタグ付与しない。
  - つまりタグ付与は `include_attendance_tag=True` かつ `decision.kind != "full"` のときのみ。
- パフォーマンス/外部API最適化:
  - `include_attendance_tag=False` の実行では、勤怠タグID取得 (`_resolve_attendance_tag_id`) をスキップする。

## テスト観点

- 入力解析:
  - Enter で `False`
  - `y` / `yes` で `True`
  - `n` / `no` で `False`
  - 不正文字列で `None`（上位でメニューエラー終了）
- 処理分岐:
  - `include_attendance_tag=False` で `put_attendance_tags` が呼ばれない。
  - `include_attendance_tag=True` で `put_attendance_tags` が呼ばれる（`full` 以外）。
- 回帰:
  - `full` 日は `include_attendance_tag=True` でもタグ付与しない。
